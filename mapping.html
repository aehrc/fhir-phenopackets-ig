Mapping

  • Building Blocks
  • Top-level Elements

This page describes the mapping between the elements in Phenopackets and the resources, profiles and extensions in the Phenopackets on FHIR implementation guide. The mapping was done by looking for the best matching FHIR resources and adding extensions to cover any gaps that were found.

Building Blocks

Phenopackets element Comment
Age Not needed because FHIR has its own age data type.
AgeRange Not needed because FHIR has its own range data type.
Biosample Corresponds to the Biosample profile.
Disease Corresponds to the Disease profile.
Evidence Corresponds to the Evidence extension.
ExternalReference Corresponds to the External Reference extension.
Gene Corresponds to ObsRegionStudied in the FHIR Genomics Reporting Implementation Guide.
HtsFile Corresponds to the Hts File profile.
Individual Corresponds to the Individual profile.
Karyotypic sex Corresponds to the individual karyotypic sex extension.
Metadata Not needed because FHIR already has its own metadata infrastructure.
OntologyClass Not needed because FHIR already has comprehensive support for terminologies.
Pedigree Corresponds to the Pedigree Node profile.
PhenotypicFeature Corresponds to the Phenotypic Feature profile.
Procedure Corresponds to Specimen.collection.
Resource Not needed because FHIR already has comprehensive support for terminologies.
Sex Not needed because FHIR already has the gender attribute in the Patient resource.
Variant Corresponds to Variant in the FHIR Genomics Reporting Implementation Guide.

Top-level Elements

Phenopackets element Comment
Phenopacket Corresponds to the Phenopackets profile.
Family Corresponds to the Family profile.
Cohort Corresponds to the Cohort profile.
Interpretation Corresponds to the Interpretation profile.
